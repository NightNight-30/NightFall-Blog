## 1. 修复 About 页面导航和高亮

### 问题
- 点击「关于」图标跳转到首页，不是 about 页面
- 进入 about 页面后图标没有高亮

### 解决步骤

#### ✅ 步骤 1：修正导航栏 URL
**文件**: `_config.stellar.yml`
```yaml
# 修改前
- id: about
  url: /  # ❌ 错误指向首页

# 修改后
- id: about
  url: /about/  # ✅ 正确路径
```

#### ✅ 步骤 2：重命名页面文件
Hexo 要求目录下必须有 `index.md` 才能作为页面访问
```bash
mv source/about/me.md source/about/index.md
```

#### ✅ 步骤 3：添加 menu_id 实现图标高亮
**文件**: `source/about/index.md`
```markdown
---
title: About me
date: 2026-04-30 17:42:02
menu_id: about  # ✅ 新增，和导航栏 id 匹配
---
```

## 📝 新增页面通用模板

以后新增页面都按这个步骤：

### 第一步：添加导航图标（_config.stellar.yml）
```yaml
menubar:
  items:
    - id: xxx          # 页面唯一标识
      theme: '#颜色'
      icon: solar:图标名
      title: 页面名称
      url: /xxx/       # 访问路径
```

### 第二步：创建页面文件
```bash
mkdir -p source/xxx
vim source/xxx/index.md
```

### 第三步：页面头部配置（front-matter）
```markdown
---
title: 页面标题
menu_id: xxx    # ⚠️ 必须和导航栏的 id 一致！
---
页面内容...
```

---

## 🔍 原理说明

Stellar 主题导航栏高亮机制：
```
导航栏 id ←→ 页面 menu_id
   ↓             ↓
 about    ==    about   → ✅ 匹配成功，图标高亮
 post     ==    post    → ✅ 默认博客页就是这个
```

## 2. 修复导航栏 tech 和 books 页面无法访问

### 问题
- 点击「技术」「读书」图标报错：`page.tags is not iterable`
- 报错位置：`note_tags.ejs` 模板

### 根因
1. **模板 bug**：`if (page.tags)` 只检查了 truthiness，没检查是否可迭代
2. **配置冲突**：手动创建 `layout: notes` 的 `index.md` 与 Stellar 的 notebook generator 冲突
   - generator 会自动生成列表页并注入 `page.posts` 数据
   - 手动创建的页面缺少这些数据

### 解决步骤

#### ✅ 步骤 1：修复主题模板 bug
**文件**: `node_modules/hexo-theme-stellar/layout/_partial/main/notebook/note_tags.ejs`
```ejs
// 修改前
<% if (page.tags) { %>

// 修改后
<% if (page.tags && page.tags.length && Array.isArray(page.tags)) { %>
  ...
  <% if (tag) { %>  <!-- 新增空检查 -->
    <a class="tag" ...><%= tag.name %></a>
  <% } %>
```

#### ✅ 步骤 2：修正 notebook base_dir
**文件**: `source/_data/notebooks/tech-notes.yml`
```yaml
# 修改前
base_dir: tech-notes

# 修改后（和导航栏 url 匹配）
base_dir: tech
```

**文件**: `source/_data/notebooks/book-notes.yml`
```yaml
base_dir: books
```

#### ✅ 步骤 3：删除冲突的 index.md
```bash
rm source/tech/index.md source/books/index.md
```
⚠️ 这些页面由 Stellar 的 generator 自动生成，不需要手动创建

#### ✅ 步骤 4：移动笔记到正确位置
Stellar 的 notebook 功能需要笔记是 **page**（不是 post）：
```bash
# 从 _posts 移到对应目录（作为 page 处理）
source/_posts/tech-notes/*.md → source/tech/
source/_posts/book-notes/*.md → source/books/
```

### 验证
```bash
npx hexo clean && npx hexo generate
# ✅ 43 files generated，无任何错误
```

## 📚 Notebook 功能正确用法

### 第一步：创建 notebook 配置
**文件**: `source/_data/notebooks/xxx.yml`
```yaml
id: xxx-notes
name: 笔记本名称
base_dir: xxx      # ⚠️ 必须和导航栏 url 一致！
menu_id: xxx       # ⚠️ 必须和导航栏 id 一致！
per_page: 20
order_by: -updated
```

### 第二步：添加导航（_config.stellar.yml）
```yaml
menubar:
  items:
    - id: xxx
      url: /xxx/    # ⚠️ 和 base_dir 一致
      ...
```

### 第三步：创建笔记（不要建 index.md！）
```bash
mkdir source/xxx
vim source/xxx/第一篇笔记.md
```
**每篇笔记 front-matter：**
```markdown
---
title: 笔记标题
notebook: xxx-notes  # ⚠️ 必须和 notebook id 一致！
menu_id: xxx         # ⚠️ 导航高亮
tags: [标签1, 标签2]
---
笔记内容...
```

### ⚠️ 重要：不要创建 index.md
Stellar 的 notebook generator 会自动生成：
- `/xxx/` - 笔记列表页
- `/xxx/tags/xxx/` - 标签过滤页
- 每篇笔记的详情页

## 3. 修复主页「近期发布」不显示笔记

### 问题
- 主页只显示 `_posts/` 下的文章（post）
- 技术笔记、读书笔记是 page，不在 `site.posts` 里
- 所以「最近更新」只看到 Hello World，看不到笔记

### 解决步骤
**文件**: `node_modules/hexo-theme-stellar/layout/_partial/widgets/recent.ejs`
```javascript
// 修改前（只显示 posts）
} else {
  arr = site.posts.filter( p => p.title && p.title.length > 0)
  arr = arr.sort("updated", -1)
}

// 修改后（posts + notebook pages 合并）
} else {
  arr = site.posts.filter(p => p.title).toArray()
  arr = arr.concat(site.pages.filter(p => p.notebook && p.title).toArray())
  arr.sort((a, b) => b.updated - a.updated)
}
```

### 效果
主页「最近更新」现在按更新时间混合显示：
- ✅ 普通博客文章（_posts/）
- ✅ Notebook 笔记（技术、读书）
